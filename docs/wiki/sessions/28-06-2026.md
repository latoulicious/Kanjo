# Summary

Mobile-browser responsiveness pass on the SPA. The shell was already
mobile-first (drawer nav, `overflow-x-auto` tables, responsive metric/chart
grids, `max-h-[90vh]` dialogs); only two real gaps remained, both fixed with
Tailwind class changes — no behavior, contract, or logic change.

# Files Touched

- `web/src/features/dashboard/DashboardPage.tsx` — `Metric` value
  `text-lg` → `text-base sm:text-lg`.
- `web/src/features/reports/ReportsPage.tsx` — same `Metric` value shrink.
- `web/src/features/transactions/TransactionDialog.tsx` — paired-field row
  `grid-cols-2` → `grid-cols-1 sm:grid-cols-2`.
- `web/src/features/transactions/TransferDialog.tsx` — same.
- `web/src/features/recurring/RecurringDialog.tsx` — same.

# Previous Behavior

- Metric cards rendered amounts at `text-lg` mono in a 2-column grid; long
  IDR values (e.g. `Rp 1.234.567.890`) overflowed the card on ~320–375px
  viewports, risking horizontal page scroll.
- Dialog paired fields (date + select / two amounts) stayed two-up below
  `sm`, squeezing native inputs on narrow phones.

# New Behavior

- Metric amounts shrink to `text-base` under `sm` (≥640px restores `text-lg`),
  so realistic amounts fit the 2-col grid; desktop unchanged.
- Dialog paired fields stack to one column under `sm`, two-up at ≥640px.

# Reason For Change

General mobile-responsiveness pass. Smallest safe diff that removes the two
overflow/cramping gaps without touching the table-first design (tables already
scroll horizontally per the design system).

# Risks

None material. Presentation-only Tailwind classes; `sm:` prefixes restore the
exact desktop layout. No DTO/API/auth/migration touched. `tsc` clean.

# Notes

Deliberately skipped table→stacked-card conversion (broad rewrite; horizontal
scroll already covers it and `design-system.md` keeps tables first-class).
