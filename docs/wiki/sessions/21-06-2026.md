# Summary

P1 — code skeleton. Implemented the `api/` Go walking skeleton (stdlib
`net/http`): compiles, boots **without a live DB** (lazy `pgxpool`), serves
`GET /health` reporting DB reachability, graceful shutdown. Mirrors the Atelier
sibling idiom (image-svc/mail-svc): `run() error`, `config.Load(os.Getenv)`,
`slog`, `signal.NotifyContext`, `http.Server` + `Shutdown`. Scope confirmed with
the user as **api-only** (no web/compose/migrations content).

# Files Touched

- `api/go.mod`, `api/go.sum` (new) — module `github.com/latoulicious/kanjo/api`,
  go 1.26; only dep is `pgx/v5`.
- `api/cmd/api/main.go` (new) — `main()`→`run()`: config → store → mux →
  `http.Server` → graceful shutdown (`shutdownTimeout` 10s).
- `api/internal/config/config.go` (new) — `Load(os.Getenv)`: `PORT` (default
  3000), `DATABASE_URL` (required).
- `api/internal/store/store.go` (new) — lazy `pgxpool` (no boot ping), `Ping()`
  for health, `Close()`.
- `api/internal/server/router.go` (new) — `ServeMux`: `GET /health` (200 up /
  503 down); `/api/v1` attach point (no routes yet).
- `api/sqlc.yaml`, `api/migrations/.gitkeep`, `api/internal/store/queries/.gitkeep`
  (new) — inert until the first migration.
- `docs/wiki/findings.md`, `resolutions.md` — F-001/F-002 ↔ R-001/R-002.
- `docs/wiki/decisions/0001-postgres-shared-instance.md` (new) — ADR.
- `docs/wiki/deploy.md` — drift banner → ADR-0001.

# Previous Behavior

No code — docs-only scaffold. `api/` did not exist.

# New Behavior

`api/` boots and serves `GET /health` (200 `{"db":"up"}` / 503
`{"db":"degraded"... "down"}`), survives an unreachable DB at boot, and shuts
down gracefully on SIGINT/SIGTERM. `/api/v1` mux is wired but carries no routes.

# Reason For Change

Approved P1 task: stand up the api walking skeleton (the skeleton reverted last
session), as the base for migrations + handlers in later phases.

# Decisions

- **CodeRabbit review** (`--agent --base main`): 2 low findings.
  - **F-001** PORT not range-checked → **wontfix** (R-001): an unbindable port
    already fails fast at `ListenAndServe`; matches sibling convention.
  - **F-002** missing `Read`/`WriteTimeout` → **fixed** (R-002): added
    `ReadTimeout`/`WriteTimeout` 30s.
- **ADR-0001 — shared Postgres**: Kanjo targets the external `shared-postgres`
  instance (network `shared-db`), dedicated db+role, **no bundled postgres
  container** → 2-container deploy. Confirmed against Lazyscan. `deploy.md`'s
  3-container model is superseded (drift banner added; full rewrite deferred to
  the deploy phase).

# Risks

- Skeleton only: no migrations/sqlc queries; `sqlc.yaml` + `queries/` are inert.
- `/health` returns 503 when the DB is down — a future compose healthcheck must
  read that as not-ready, not as a crash.
- `deploy.md` is now partially stale (pointed at ADR-0001 until rewritten).
