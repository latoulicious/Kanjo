# Principles

Source: the Kanjo plan (`Kanjo.pdf`). This file is the product's spine — every
feature is measured against it.

## Summary

Kanjo (勘定, "account / reckoning") is a personal financial ledger built for
builders, self-hosters, and engineers. It treats money as a **resource system**
rather than a collection of expenses, and focuses on the operational state of
finances: monthly burn, savings rate, runway, cash flow, and project costs.

Finance is organized around **operations** — housing, transportation, food,
infrastructure, projects, hobbies.

## Philosophy

- **Money Is Fuel** — money is a resource that powers operations, not a score.
- **Clarity Over Accounting** — understanding beats bookkeeping correctness.
- **Systems Thinking** — model flows and state, not isolated transactions.
- **Intentional Spending** — surface where resources go so choices are deliberate.
- **Operational View** — answer "is current spending sustainable?", not "what did
  I buy?".

## Guiding Principle

> Every feature should improve understanding of where resources go and how
> sustainable current spending is.

If a proposed feature does not move that understanding, it does not belong in
Kanjo.

## Scope (MVP)

dashboard · ledger · accounts · categories · projects · reports.

## Non-Goals

Kanjo is **not** a fintech platform, budgeting coach, banking application,
investment platform, or startup SaaS.

Explicitly excluded from the product: bank sync, OCR, receipt scanning, AI
features, investment tracking, budgeting systems, multi-user support, PWA
support, mobile app.
